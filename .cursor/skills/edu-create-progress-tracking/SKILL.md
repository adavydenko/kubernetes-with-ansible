# Create Progress Tracking System

Create comprehensive progress tracking systems with checklists, assessment levels, recommendations, and personalization sections for self-evaluation.

## When to Use

- Creating educational materials or courses
- Need self-assessment capabilities
- User mentions "progress", "checklist", "tracking", or "self-evaluation"
- Creating learning guides that need progress monitoring
- Building training programs

## Instructions

### 1. Define Assessment Levels

Create 4-5 assessment levels:
- ❌ **Not Started** - Haven't begun
- 🔄 **In Progress** - Studying, needs practice
- ✅ **Completed** - Fully mastered
- ⭐ **Advanced** - Deep expertise

Alternative levels:
- **Beginner** (0-25%) - Learning basics
- **Intermediate** (26-50%) - Confident with basics
- **Advanced** (51-75%) - Deep knowledge
- **Expert** (76-100%) - Mastery

### 2. Structure Checklists by Section

For each learning section:
- Create specific, measurable items
- Group by topic/subtopic
- Use clear, actionable language
- Include 5-7 items per subsection

Example:
```markdown
## Section 1: Basics

### 1.1 Topic A
- [ ] Understand concept X
- [ ] Can explain Y
- [ ] Can apply Z
- [ ] ⭐ Advanced: Can optimize Z
```

### 3. Add Practical Skills

Include hands-on skills:
- "Can deploy X"
- "Can configure Y"
- "Can troubleshoot Z"
- "Can optimize W"

### 4. Create Recommendations

Based on assessment level:
- **Mostly ❌**: Start with basics, practice simple examples
- **Mostly 🔄**: Deepen practice, solve more tasks, experiment
- **Mostly ✅**: Move to advanced topics, prepare for certification
- **Mostly ⭐**: Share knowledge, participate in community, create projects

### 5. Add Personalization Sections

Include:
- **Goals**: Personal learning goals
- **Areas for Improvement**: What to focus on
- **Achievements**: Track successes
- **Notes**: Personal observations

### 6. Set Regular Review Schedule

Recommend frequency:
- **Weekly**: Track progress
- **Monthly**: Adjust goals
- **Quarterly**: Plan development

## Best Practices

- ✅ Use multiple assessment levels (not just yes/no)
- ✅ Create specific, measurable items
- ✅ Group by topics/sections
- ✅ Include practical skills
- ✅ Provide recommendations based on level
- ✅ Add personalization sections
- ✅ Recommend regular review schedule
- ✅ Make it motivating and visual

## Checklist Structure

### Basic Structure
```markdown
## Section Name

### Subsection 1.1
- [ ] Item 1
- [ ] Item 2
- [ ] ⭐ Advanced item

### Subsection 1.2
- [ ] Item 1
- [ ] Item 2
```

### With Assessment Levels
```markdown
## Section Name

### Subsection 1.1
- [ ] Item 1 - ❌ Not Started
- [ ] Item 2 - 🔄 In Progress
- [ ] Item 3 - ✅ Completed
- [ ] ⭐ Advanced item - ⭐ Advanced
```

## Assessment Level System

### 4-Level System
```markdown
- ❌ **Not Started** - Haven't begun
- 🔄 **In Progress** - Studying, needs practice
- ✅ **Completed** - Fully mastered
- ⭐ **Advanced** - Deep expertise
```

### Percentage-Based System
```markdown
- [ ] **Beginner** (0-25%) - Learning basics
- [ ] **Intermediate** (26-50%) - Confident with basics
- [ ] **Advanced** (51-75%) - Deep knowledge
- [ ] **Expert** (76-100%) - Mastery
```

## Recommendations Template

### For Beginners (Mostly ❌)
```markdown
**Рекомендация:** Начните с основ (Разделы 1-2), 
практикуйтесь с простыми примерами
```

### For Learners (Mostly 🔄)
```markdown
**Рекомендация:** Углубитесь в практику, решайте 
больше задач, экспериментируйте
```

### For Advanced (Mostly ✅)
```markdown
**Рекомендация:** Переходите к продвинутым темам, 
готовьтесь к сертификации
```

### For Experts (Mostly ⭐)
```markdown
**Рекомендация:** Делитесь знаниями, участвуйте 
в сообществе, создавайте собственные проекты
```

## Complete Structure Example

```markdown
# Progress Checklists

## How to Use
[Instructions]

## Section 1: Basics
### 1.1 Topic A
- [ ] Item 1
- [ ] Item 2
- [ ] ⭐ Advanced item

## Overall Progress
### Basic Skills (Sections 1-4)
- [ ] Beginner (0-25%)
- [ ] Intermediate (26-50%)
- [ ] Advanced (51-75%)
- [ ] Expert (76-100%)

## Recommendations
[Based on assessment level]

## Personal Notes
### Goals
- [ ] Goal 1

### Areas for Improvement
- [ ] Area 1

## Regular Review
**Last check:** [Date]
**Next check:** [Date]
```

## What NOT to Do

- ❌ Don't use only yes/no (use multiple levels)
- ❌ Don't create vague items
- ❌ Don't skip recommendations
- ❌ Don't forget personalization
- ❌ Don't make it too complex
- ❌ Don't skip regular review schedule
