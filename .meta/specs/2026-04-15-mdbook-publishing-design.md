# Design Spec: Публикация учебного пособия через mdBook

## Статус

- **Дата:** 2026-04-15
- **Проект:** `kubernetes-with-ansible`
- **Решение:** `mdBook` как целевой движок публикации
- **Автор:** AI assistant + user alignment

## Контекст

В репозитории уже есть готовая база Markdown-материалов в `docs/` (теория, практикум, упражнения, квизы, reference). Требуется опубликовать материалы как:

1. Интерактивный веб-учебник (book-like UX).
2. Печатный/распространяемый PDF-документ (book export).

Ключевое ограничение: не переписывать материалы в LaTeX и сохранить Markdown как единственный источник.

## Цели

- Использовать существующие `.md` файлы как основу без полной переработки.
- Получить "книжный" web UX с быстрой навигацией и поиском.
- Сохранять единый source-of-truth для web и PDF.
- Настроить простую публикацию на GitHub Pages.

## Не-цели

- Полноценная издательская верстка уровня индивидуального типографического шаблона на первом этапе.
- Полный редизайн всех текстов и переписывание структуры курса с нуля.
- Внедрение тяжелой CMS-платформы.

## Выбранный подход

Выбрать `mdBook` как основной web-движок и собрать контент по книге через `SUMMARY.md`.

- **Web:** `mdbook build` -> статический сайт (`book/`) -> GitHub Pages.
- **PDF:** отдельный шаг экспорта из того же Markdown-контента (итеративно: сначала базовый, затем улучшенный quality pipeline).

Такой подход дает лучший "книжный" UX при минимальной миграции.

## Альтернативы (кратко)

- **Quarto:** сильнее в PDF из коробки, но user preference смещен к визуальному стилю mdBook.
- **Hugo + темы:** выше гибкость дизайна, но выше трудозатраты и сложнее удерживать единый pipeline для PDF.
- **Docusaurus/MkDocs:** сильные docs-платформы, но book-like восприятие слабее из коробки.

## Архитектура решения

### 1) Контентный слой

- Исходники остаются в `docs/`.
- Добавляется curated оглавление в `SUMMARY.md` (порядок глав, иерархия разделов).
- Минимальная нормализация markdown:
  - единая иерархия заголовков;
  - проверка ссылок и относительных путей;
  - упрощение нестабильных конструкций.

### 2) Рендеринг web

- `mdbook build` генерирует сайт-книгу с:
  - левым оглавлением;
  - полнотекстовым поиском;
  - чистой chapter-навигацией;
  - базовой theming-настройкой.

### 3) PDF-ветка

PDF делается из того же Markdown источника, без переписывания контента:

- Фаза 1: базовый PDF export для распространения.
- Фаза 2: улучшение типографики, титульных данных, оглавления и нумерации.

Технический инструмент PDF выбирается отдельно в implementation-plan (критерий: стабильность на Windows и CI).

### 4) Доставка

- GitHub Actions:
  - build web book;
  - deploy на GitHub Pages;
  - build PDF и публикация артефакта (release artifact).

## Поток данных и артефакты

1. Markdown в `docs/` -> curated `SUMMARY.md`.
2. `mdBook` -> HTML site (`book/`).
3. Pipeline deploy -> GitHub Pages.
4. Pipeline export -> `pdf` artifact для скачивания.

## Error Handling и риски

- **Broken links/anchors:** добавить link-check в CI.
- **Нестабильный PDF export:** сохранить fallback путь (временный базовый exporter).
- **Разъезд web/PDF структуры:** использовать единый master-оглавление как источник порядка глав.
- **Рост ручных правок:** фиксировать style guide для markdown-авторов.

## Testing & Verification

- Локально:
  - сборка `mdbook build`;
  - визуальная проверка 5-10 ключевых страниц;
  - smoke-тест поиска и навигации.
- CI:
  - успешная сборка;
  - проверка ссылок;
  - наличие итогового `book/` и PDF артефакта.

## Этапы внедрения

### Этап A: Быстрый web launch

- Инициализировать `mdBook` структуру.
- Подключить существующие главы через `SUMMARY.md`.
- Собрать и задеплоить GitHub Pages.

### Этап B: UX polish

- Подчистить заголовки/ссылки/навигацию.
- Настроить тему и типографику под учебный стиль.
- Упорядочить разделы для линейного обучения.

### Этап C: PDF quality

- Подключить стабильный PDF exporter.
- Добавить базовые академические элементы (титульные метаданные, TOC, номера разделов).
- Проверить читаемость и печатный формат.

## Критерии приемки

- Сайт открывается на GitHub Pages и покрывает выбранный набор учебных разделов.
- Навигация и поиск работают без критичных ошибок.
- PDF генерируется из того же Markdown источника.
- Для обновления контента требуется только редактирование `.md` и повторная сборка.

## Решение, зафиксированное с пользователем

- Целевой framework: **mdBook**.
- Приоритет: **book-like UX в web**.
- Доп. требование: **PDF из того же контента**, без переписывания всего материала в LaTeX.
